# 考生数据统计页面设计

## 背景

OP_ADMIN（系统管理员）需要一个独立的考生数据统计页面，支持多维度分析全平台各院校的考生数据。

## 统计维度

| 维度 | 数据来源 | 说明 |
|------|---------|------|
| 性别 | `gender` 字段 | 男/女/其他 分布比例 |
| 年龄 | `birth_date` 计算 | 按年龄段：<20, 20-25, 26-30, 30+ |
| 录取轮次 | `push_round` 字段 | 各轮次的考生数量分布 |
| 推送次数 | 按 `candidate_id` 去重统计 | 被推送1/2/3/...次的考生人数 |
| 院校录取区间 | 各院校 ADMITTED/CONDITIONAL 状态计数 | 按区间分组统计 |

### 院校录取区间

| 区间 | 说明 |
|------|------|
| 0-10人 | 小型院校 |
| 11-30人 | 中小型 |
| 31-50人 | 中型 |
| 51-100人 | 大型 |
| 100+人 | 超大型 |

## API 设计

### GET /api/v1/statistics/students

查询参数：
- `schoolId` (可选, OP_ADMIN筛选院校)
- `round` (可选, 录取轮次)
- `startDate` / `endDate` (可选, 推送时间范围)

响应：
```json
{
  "totalCandidates": 1234,
  "genderDistribution": { "male": 740, "female": 490, "other": 4 },
  "ageDistribution": { "<20": 100, "20-25": 800, "26-30": 300, "30+": 34 },
  "roundDistribution": { "1": 500, "2": 400, "3": 334 },
  "pushCountDistribution": { "1": 800, "2": 300, "3plus": 134 },
  "schoolAdmissionRanges": [
    { "range": "0-10", "schoolCount": 150, "admissionCount": 800 },
    { "range": "11-30", "schoolCount": 80, "admissionCount": 1600 },
    { "range": "31-50", "schoolCount": 40, "admissionCount": 1500 },
    { "range": "51-100", "schoolCount": 20, "admissionCount": 1400 },
    { "range": "100+", "schoolCount": 10, "admissionCount": 1500 }
  ]
}
```

### GET /api/v1/statistics/students/export

导出 Excel 文件，包含所有统计维度的详细数据。

## 实现文件

### 后端
- `controller/StatisticsController.java` - 统计 API
- `service/StatisticsService.java` - 统计业务逻辑
- `repository/CandidatePushRepository.java` - 数据查询

### 前端
- `views/admin/StatisticsView.vue` - 统计页面
- `api/statistics.js` - API 调用
- `router/index.js` - 路由配置

## 验证方式

1. 启动后端和前端
2. 以 OP_ADMIN 登录
3. 访问统计页面，验证各维度数据正确
4. 测试导出功能
5. 测试筛选条件（院校、轮次、时间范围）
