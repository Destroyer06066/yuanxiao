# 院校管理平台 — Docker 生产部署指南

## 服务器要求

| 资源 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 2 核 | 4 核+ |
| 内存 | 4 GB | 8 GB+ |
| 磁盘 | 50 GB SSD | 100 GB SSD+ |
| 系统 | Ubuntu 22.04 LTS | Ubuntu 22.04 LTS |

## 部署架构

```
                    ┌─────────────────────────────────┐
Internet ───► Nginx (80/443) ──► Frontend (Nginx)
                                     │
                                     ▼
                               Backend (Spring Boot)
                                     │
                         ┌───────────┼───────────┐
                         ▼           ▼           ▼
                    PostgreSQL     Redis       MinIO
```

## 部署步骤

### 1. 服务器准备

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Docker & Docker Compose
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER
newgrp docker

# 验证
docker --version
docker-compose --version
```

### 2. 创建项目目录

```bash
sudo mkdir -p /opt/campus-platform
sudo chown $USER:$USER /opt/campus-platform
cd /opt/campus-platform
```

### 3. 上传代码

```bash
# 上传代码到 /opt/campus-platform/
# 或使用 git clone
git clone <repo> .
```

### 4. 配置环境变量

```bash
cp .env.example .env
nano .env
```

**必须修改的配置项：**

```env
# 数据库（必须）
DB_PASSWORD=你的强密码

# Redis
REDIS_PASSWORD=你的Redis密码

# MinIO（必须）
MINIO_ACCESS_KEY=你的MinIOAccessKey
MINIO_SECRET_KEY=你的MinIOSecretKey
MINIO_PUBLIC_URL=https://你的域名/minio

# JWT 密钥（生产必须使用 RSA 密钥对）
# 先生成密钥对：
openssl genrsa -out jwt-private.pem 2048
openssl rsa -in jwt-private.pem -pubout -out jwt-public.pem
mv jwt-private.pem backend/src/main/resources/keys/
mv jwt-public.pem backend/src/main/resources/keys/

# 报名平台（正式环境填真实地址）
REGISTRATION_PLATFORM_URL=https://报名平台真实地址/api
```

### 5. 数据库初始化

```bash
# 启动数据库
docker-compose up -d postgres

# 等待数据库就绪
sleep 5

# Flyway 会在后端首次启动时自动执行迁移
```

### 6. 构建并启动

```bash
# 开发/测试环境（单台机器快速部署）
docker-compose -f docker/docker-compose.dev.yml up -d

# 生产环境
docker-compose -f docker/docker-compose.prod.yml up -d --build
```

### 7. 验证服务

```bash
# 检查容器状态
docker-compose -f docker/docker-compose.prod.yml ps

# 检查健康状态
curl http://localhost:8080/actuator/health
curl http://localhost/api/v1/auth/login  # 应返回401

# 查看日志
docker-compose -f docker/docker-compose.prod.yml logs -f backend
```

## Nginx HTTPS 配置

生产环境必须启用 HTTPS。以下是 Let's Encrypt 免费证书配置：

```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书（需先配置好域名 DNS）
sudo certbot --nginx -d your-domain.com
```

## 数据备份策略

### PostgreSQL 备份

```bash
# 每日凌晨 3 点备份（加入 crontab）
0 3 * * * docker exec campus_postgres_prod pg_dump -U postgres campus_platform > /backup/campus_$(date +\%Y\%m\%d).sql
```

### Redis 备份

```bash
# RDB 持久化（已默认开启，存于 docker/redis_data/）
# 建议定期备份 docker volumes
docker run --rm -v campus-platform_redis_data:/data -v /backup:/backup alpine tar czf /backup/redis_$(date +\%Y\%m\%d).tar.gz -C /data .
```

### MinIO 数据备份

```bash
# 使用 MinIO Client
mc mirror minio/campus-exports /backup/minio-$(date +\%Y\%m\%d)/
```

## 更新部署

```bash
cd /opt/campus-platform

# 拉取新代码
git pull

# 重新构建并重启
docker-compose -f docker/docker-compose.prod.yml up -d --build backend
```

## 故障排查

| 问题 | 排查命令 |
|------|---------|
| 后端启动失败 | `docker-compose logs backend` |
| 数据库连接失败 | `docker-compose logs postgres` |
| 前端无法访问 | `docker-compose logs frontend` |
| JWT Token 无效 | 检查 Redis `session:*` key 是否存在 |
| 名额乐观锁冲突 | 检查 `admission_quota.version` 是否正确递增 |
