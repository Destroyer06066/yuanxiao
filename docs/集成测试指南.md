# 集成测试指南

## 本地开发环境启动

### 1. 启动基础设施
```bash
cd /Volumes/HP\ P900/Tech/yuanxiao/campus-platform/docker
docker-compose up -d postgres redis minio mock-server
```

### 2. 启动后端
```bash
cd backend
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```
Swagger UI: http://localhost:8080/swagger-ui.html

### 3. 启动前端
```bash
cd frontend
npm install
npm run dev
```
访问: http://localhost:5173

## 核心流程测试用例

### TC-01: 登录
1. POST `/api/v1/auth/login`，账号 `op_admin`，密码 `OpAdmin@2026`
2. 预期：返回 `accessToken`，`role=OP_ADMIN`
3. 后续请求 Header 携带 `Authorization: Bearer <token>`

### TC-02: 创建院校
1. 以 OP_ADMIN 身份登录
2. POST `/api/v1/admin/schools`
```json
{
  "schoolName": "北京语言大学",
  "schoolShortName": "北语",
  "province": "北京市",
  "schoolType": "综合类",
  "contactName": "王老师",
  "contactPhone": "13800138000",
  "contactEmail": "wang@blcu.edu.cn"
}
```
3. 预期：`code: 0`，返回 `schoolId`

### TC-03: 创建院校管理员
1. POST `/api/v1/admin/schools/{schoolId}/admin-account`
```json
{
  "username": "blcu_admin",
  "realName": "王招生",
  "phone": "13800138001"
}
```
2. 预期：返回初始密码（12位），仅展示一次

### TC-04: 接收考生推送（Mock）
1. POST `/api/v1/integration/push`
```json
{
  "candidateId": "CAND-2026-001",
  "candidateName": "张三",
  "nationality": "哈萨克斯坦",
  "totalScore": 268.5,
  "subjectScores": { "数学": 92, "物理": 88, "化学": 88.5 },
  "intention": "计算机科学",
  "schoolId": "<创建好的院校UUID>",
  "round": 0
}
```
2. 预期：`code: 0`，`pushId` 返回

### TC-05: 查询考生列表
1. GET `/api/v1/students?page=1&pageSize=20`
2. 预期：返回考生列表，包含姓名、分数、状态

### TC-06: 直接录取
1. POST `/api/v1/students/admit`
```json
{
  "pushId": "<TC-04返回的pushId>",
  "majorId": "<创建好的专业UUID>",
  "remark": "优秀考生，直接录取"
}
```
2. 预期：`code: 0`，考生状态变为 `ADMITTED`

### TC-07: 批量拒绝
1. POST `/api/v1/students/batch-reject`
```json
{ "pushIds": ["<pushId1>", "<pushId2>"] }
```

### TC-08: 报名平台 Mock 接口测试
1. POST `http://localhost:8081/mock-api/admission-notify`
```json
{ "schoolId": "...", "candidateId": "...", "candidateName": "张三" }
```
2. 预期：返回 `{"code":0}`

### TC-09: 触发考生确认（互斥测试）
1. 考生先在院校A被录取（ADMITTED），再确认院校A录取
2. POST `/api/v1/integration/candidate-confirmed`
```json
{
  "candidateId": "CAND-2026-001",
  "confirmedSchoolId": "<院校A的UUID>",
  "confirmedAt": "2026-03-26T10:00:00Z"
}
```
3. 预期：院校A考生状态 → `CONFIRMED`

### TC-10: 有条件录取 + 到期
1. 有条件录取考生 → 状态 `CONDITIONAL`
2. 等待定时任务扫描（每10分钟），或手动 POST `/api/v1/integration/condition-expired`
3. 预期：状态自动恢复为 `PENDING`，名额释放

## JWT 续期测试
1. 登录获取 token
2. 不操作，等待 30 分钟
3. 发起任意请求
4. 预期：响应头 `X-New-Token` 包含新 token

## 并发测试（名额乐观锁）
使用 JMeter 或以下脚本模拟 10 并发录取同一专业：
```bash
for i in {1..10}; do
  curl -X POST http://localhost:8080/api/v1/students/admit \
    -H "Authorization: Bearer <token>" \
    -H "Content-Type: application/json" \
    -d '{"pushId":"<uuid>","majorId":"<uuid>"}' &
done
wait
```
预期：9 个成功，1 个因名额不足失败（乐观锁冲突）
